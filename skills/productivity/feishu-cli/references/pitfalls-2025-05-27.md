# 飞书 CLI 踩坑记录（2025-05-27）

## 问题1：OAuth 反复授权

**现象**：每次 `--scope` 只能申请一个权限，用户需要反复扫码。

**解决**：用 `--recommend` 一次性获取所有推荐权限。

```bash
lark-cli auth login --no-wait --json --recommend
```

## 问题2：docs +create --content 内容写入失败（✅ 已解决）

**现象**：创建文档时传入 `--content "字符串"` 或 `--content @file.md`，文档创建成功但内容为空。

**错误码**：`degrade_code=1011, msg=Instruction produced no document changes`

**根因**：`--content` 参数不支持字符串或文件路径，必须用 stdin（`-`）配合 heredoc。

**❌ 错误方式**：
```bash
lark-cli docs +create --title "标题" --wiki-space "xxx" --content "# 标题\n\n内容" --api-version v2
lark-cli docs +create --title "标题" --wiki-space "xxx" --content @file.md --api-version v2
```

**✅ 正确方式**：
```bash
lark-cli docs +create --title "标题" --wiki-space "xxx" \
  --content - --doc-format markdown --api-version v2 << 'EOF'
# 标题

内容
EOF
```

**关键**：
- `--content -` 表示从 stdin 读取
- `<< 'EOF'` heredoc 提供真正换行（EOF 带引号防止变量展开）
- `--doc-format markdown` 明确指定 Markdown 格式

**已验证支持**：标题、列表、表格、代码块、引用、Todo 勾选框等完整 Markdown 语法。

## 问题3：config.json 引号问题

**现象**：`lark-cli config bind` 后，config.json 中 appId 和 brand 有多余的单引号。

**解决**：手动编辑 `~/.lark-cli/hermes/config.json`，去掉多余引号。

## 问题4：个人空间 vs 团队空间

**现象**：应用 API 无法访问用户的个人知识库（my.feishu.cn/wiki/xxx）。

**原因**：飞书 API 对个人空间和团队空间有隔离，应用只能访问团队空间。

**解决**：
- 方案A：用户把应用添加到个人知识库成员中（通常无效）
- 方案B：创建团队知识库，应用自动有权限
- 方案C：用云空间文件夹替代知识库

## 问题5：device code 过期

**现象**：用户授权后，用 device-code 完成登录时提示 `device_code has expired`。

**原因**：device code 有效期只有 10 分钟。

**解决**：重新生成 `lark-cli auth login --no-wait --json --recommend`。

## 已验证可用的操作

### ✅ 创建知识库文档（带内容）
```bash
lark-cli docs +create --title "标题" --wiki-space "xxx" \
  --content - --doc-format markdown --api-version v2 << 'EOF'
# 标题

内容
EOF
```

### ✅ 更新文档内容
```bash
lark-cli docs +update --doc "xxx" --command overwrite \
  --content - --doc-format markdown --api-version v2 << 'EOF'
新内容
EOF
```

### ✅ 创建知识库文档节点（空文档）
```bash
lark-cli wiki +node-create --space-id "xxx" --title "标题" --obj-type docx
```

### ✅ 列出知识库空间
```bash
lark-cli wiki +space-list
```

### ✅ 读取文档
```bash
lark-cli docs +fetch --doc "xxx" --api-version v2
```

### ✅ 创建云盘文件夹
```bash
lark-cli drive +create-folder --name "文件夹" --parent "xxx"
```
